几道单调栈的使用的题目，都是找后面的最近元素的问题。

Ⅰ

题目大意

LeetCode 496，给定两个没有重复元素的数组，其中 nums1 是 nums2 的子集，在 nums2 中找到 nums1 中的每个元素的下一个较大的元素，没有就置为 -1。

解题思路

既然数组1是数组2的子集，直接将数组2全部算完之后存入 HashMap 中，然后遍历数组1查询。

数组2中，维护一个栈用于存放还没找到较大值的元素位置，那么这个栈其实就是降序栈，栈顶是最近一个没找到较大值的值。这个栈通过遍历的时候维护，同时遍历的时候观察当前元素是不是大于栈顶的元素，是的话就往前将所有符合条件的值都弹出，不管是不是当前的值都要入栈。

后面的 LeetCode 739 就是这个思路。

Ⅱ

题目大意

LeetCode 503，给一个循环数组，返回一个等长的数组，数组中的每一个元素是：它后面的第一个大于它的元素（如果后面没有就循环一遍到最前面找，直到循环了一圈为止），如果不存在这样的数，就返回 -1。

解题思路

建立等长结果数组，初始化为-1。

先处理后面有存在比自己大的位置的值。对于每个值，将他们的下标都放入栈，放入之前看一下，如果比栈顶代表的元素大，说明这是离栈顶最近的较大元素，将栈顶位置结果记录，并弹出，考虑到这个值可能比前面的数也大，使用 while 循环，更新前面所有比当前数小的位置的结果。经过这一步处理后，每一个后面有比自己大的值的 index 都被弹出，只剩下后面找不到比自己大的值的 index 了。

开始第二次循环，从开始的数字寻找，如果栈顶元素比当前数字小，说明当前数字就是栈顶数字后面的第一个较大的数字，将栈顶数字位置结果更新并弹出。遍历完成之后，栈中只会剩下唯一一个元素，即数组中的最大值，他的结果位没有改动，依旧是 -1。

Ⅲ

题目大意

整数的下一个相同元素排列的较大的数。

解题思路

和 Next Per 全排列的思路一样，从后往前先找到开始降序的那个较小的数，同时把这个数保存下来。然后从后往前找第一个比找到的数大的数，交换位置后，把后面的数反转一下。

这里因为是整数，所以涉及到重新拼装的事，将所有的数拆分三部分，第一部分是不动的，第二部分是那个换过来的数，第三部分是反转后的数。
